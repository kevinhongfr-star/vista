## TASK 2: Discount Rules Engine (R-03 + R-18)

**Goal:** API to validate discount requests + enforce never-discount rules. Frontend component.

### API Routes to create:

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/discount-rules` | List all active rules |
| POST | `/api/discount-rules/check` | Validate discount request |

### POST `/api/discount-rules/check`:
**Input:**
```json
{
  "service_id": "uuid",
  "requested_discount_pct": 20,
  "contact_id": "uuid (optional)"
}
```

**Logic:**
1. Look up service → get tier
2. Find applicable rules for that tier in `vista_discount_rules`
3. Check if `never_override = true` for that tier → immediate reject
4. Check condition_type + condition_params for any special conditions (e.g., `founding_client`)
5. Return max allowed discount

**Output:**
```json
{
  "allowed": true,
  "max_allowed_pct": 15,
  "frame_as": "Early-bird package",
  "reason": "Tier 2 services allow up to 15% with early-bird framing"
}
```

### Hard rules (from seed data):
- **Search (tier 5):** `never_override = true` → always `allowed: false`
- **Platform (tier 7):** `never_override = true` → always `allowed: false`
- **Diagnostic with founding_client condition:** up to 50% off
- **Post-founding retainer:** `never_override = true`

### Frontend component:
`<DiscountField serviceId={id} />` 
- Shows discount % input
- Auto-caps at max allowed %
- Shows red warning banner for non-discountable services
- Shows "frame as" suggestion text

### Acceptance criteria:
- [ ] Search tier → cannot discount, red warning shown
- [ ] Platform tier → cannot discount
- [ ] Diagnostic with founding_client → up to 50% allowed
- [ ] Post-founding retainer → cannot discount
- [ ] UI shows `frame_as` text when discount is allowed
- [ ] `never_override` services show disabled input + warning
